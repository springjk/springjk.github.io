Docker安装教程

发表于 2016-11-30   |  

Docker 刚发布时，定位是一个单机的轻量化虚拟化工具，只被部分运维人员知晓。但伴随着 Docker 大生态的发展，Google 、微软、亚马逊等大厂的加入，Docker 逐渐为开发所知，于是广大程序员发现这个「码头工」的软件简直是性感无比，而且完美的阐释了「码头工」的概念，你只要打包好你的应用，「码头工」就能把它无差别的搬运到 Every where，这解决了开发和运维的很多痛点，经常切换于多种操作系统、部署管理大量应用的人应该是明明白白这个痛，当然最高兴的应该还是各个 pass 云了，可以又省一波钱了，这是资源隔离和应用沙箱的话题了，这里不展开了。

Docker 虽然不是第一个做这类事的，确是目前做的最好的，「她」将会给我们的学习、开发、部署、运维带来极大便利，你一定会爱上「她」，如果你还没爱上，那你只是在爱「她」的路上而已。

既然来看这篇文章，必然也已经大致了解过了，这里不过多讲述，让我们来一个三分钟安装吧。

安装

首先抛出 官方地址，嗯，要是去看了的应该也不会看这篇文章了，这里推荐使用国内 Daocloud 的源进行安装，地址如下：

  • Docker for Windows
  • Docker for Mac
  • Docker for Linux

系统要求：

Windows : Windows10 x64位 Build 10586以上版本（10586 已经是一年前的版本了！）。What？还是 Windows xp/7！本是同根生，相煎何太急。难道你没自己没写过兼容代码？请速速升级！冥顽不灵的就去使用 Docker Toolbox 吧，这是基于 VirtualBox 虚拟机实现的版本，使用会有诸多不便，这其中的不便就自己去承担吧， 自己的选择，自己负责，Goodluck！

Mac : OS X 10.10.3 或者更高版本 (可以忽略，基本都是)

Linux : x64 3.10 内核以后 (可以忽略，LTS 基本都是)

注意事项：

Windows 用户如果没有开启过 Hyper-V，将会提示：

Hyper-V feature is not enabled.
Do you want to enable it for Docker to be able to work properly?
Your computer will restart automatically.

确认后将会自动安装 Hyper-V 并重启，重启后安装完成。完成启动后在右下角角标找到 Docker 右键->Settings->Shared Driver，勾选上你工作区的目录，点击 Apply (这是为了在创建容器的时候挂载目录而设置的，如果你不清楚这代表什么可勾选全部盘符，以免使用中引发困惑)。

加速器

DockerHub 的镜像在国外，国内访问速度非常慢，在使用前请先配置好加速器，这里为两个公开加速器：

  • 网易加速器 ：http://hub-mirror.c.163.com
  • 中科大加速器 ：https://docker.mirrors.ustc.edu.cn
  • 另外还有 Daocloud 、Aliyun 等多家私有加速器，需要注册后使用

选择好加速器后在 Docker 设置中添加 Registry mirrors 后应用重启。

验证

打开命令行，输入 docker -v，如输出版本即安装成功。此处强烈建议 Windows 用户使用 [Cmder] (http://cmder.net) 作为命令行工具，为啥，你不得不爱上她，毕竟已经是最好的了。
